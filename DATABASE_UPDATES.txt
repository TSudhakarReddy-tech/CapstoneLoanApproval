================================================================================
NEW DATABASE FIELDS
================================================================================

Added to LoanDecisionRecord table:

1. case_id (String, Unique)
   - Formatted case identifier
   - Example: CASE-APP-20240615-A1B2C3
   - Indexed for fast lookups

2. decision_category (String)
   - APPROVED, REJECTED, or UNDER_REVIEW
   - Enables easy filtering by decision type

3. decision_summary (Text)
   - Detailed, formatted summary
   - Ready to send to applicants
   - Decision-specific content

4. rejection_reason (Text)
   - Detailed reason for rejection
   - Only populated for REJECTED decisions
   - Supports appeal process

5. approval_conditions (JSON)
   - Conditions for approval
   - Only populated for APPROVED decisions
   - Examples: interest_rate, tenure, insurance

6. review_notes (Text)
   - Notes for under-review cases
   - Reason for review
   - Required documentation notes

7. risk_level (String)
   - LOW, MEDIUM, HIGH, or CRITICAL
   - Calculated from confidence score
   - Enables risk-based monitoring

================================================================================
CASE ID FORMAT
================================================================================

CASE-{TYPE}-{YYYYMMDD}-{HASH}

Type Codes:
  APP - Approved
  REJ - Rejected
  REV - Under Review
  GEN - Generic/Other

Examples:
  CASE-APP-20240615-A1B2C3  → Approval decision dated June 15, 2024
  CASE-REJ-20240615-D4E5F6  → Rejection decision dated June 15, 2024
  CASE-REV-20240615-G7H8I9  → Under review decision dated June 15, 2024

================================================================================
DECISION SUMMARIES
================================================================================

Each decision type generates a professional, formatted summary:

1. APPROVAL SUMMARY
   - Includes loan details, applicant info, conditions
   - States validity period (30 days)
   - Professional tone suitable for applicant communication

2. REJECTION SUMMARY
   - Lists numbered rejection reasons
   - Includes appeal process
   - Provides contact information

3. UNDER REVIEW SUMMARY
   - Explains review reason
   - Lists required documentation
   - Sets expectations (5-7 business days)

================================================================================
RISK LEVEL CALCULATION
================================================================================

Confidence Score → Risk Level

  ≥ 90%        → LOW         (Safe decision)
  75% - 89%    → MEDIUM      (Moderate confidence)
  60% - 74%    → HIGH        (Low confidence)
  < 60%        → CRITICAL    (Very low confidence)

Risk levels help prioritize decisions for review and monitoring.

QUICK START
================================================================================

1. Import the utilities:
   from app.database import add_decision_with_summary, get_db_session

2. Create a decision:
   db = get_db_session()
   decision = add_decision_with_summary(
       db=db,
       application_id="app_123",
       decision_type="APPROVED",  # or "REJECTED", "UNDER_REVIEW"
       confidence_score=0.92,
       profile_analysis={...},
       financial_analysis={...},
       decision_reasoning="Meets all criteria",
       case_reference="REF-001"
   )

3. Use the result:
   print(f"Case ID: {decision.case_id}")
   print(f"Category: {decision.decision_category}")
   print(f"Risk: {decision.risk_level}")
   print(f"Summary: {decision.decision_summary}")

MIGRATION FOR EXISTING DATA
================================================================================

If you have existing decisions in your database:

from app.database import migrate_existing_decisions, get_db_session

db = get_db_session()
try:
    stats = migrate_existing_decisions(db)
    print(f"Total: {stats['total_records']}")
    print(f"Updated: {stats['updated_records']}")
    print(f"Errors: {len(stats['errors'])}")
finally:
    db.close()

This updates existing records with new fields while maintaining data integrity.
